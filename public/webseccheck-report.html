Critical Content Security Policy FAIL ‚Ä∫

Description

The Content-Security-Policy header is not set. CSP is the most powerful browser-side defense against XSS attacks.

Impact

  • Attackers can inject malicious scripts
  • No protection against inline script execution
  • Third-party resources loaded without restriction

Add CSP Header

Add a Content-Security-Policy header via your web server or CDN.

Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; font-src 'self' data:; connect-src 'self' https:; frame-ancestors 'none';
